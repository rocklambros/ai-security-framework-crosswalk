\section{Method}
\label{sec:method}

Our pipeline comprises four stages: feature extraction, stacking
classification, conformal calibration, and uncertainty routing.

\subsection{Feature Extraction}

For each candidate control pair $(u, v)$, we extract a
22-dimensional feature vector organized into three families:

\paragraph{LLM similarity features (7 dimensions).}
We compute dense semantic similarity using BGE-large (1024-d)
embeddings, yielding cosine similarity between control texts. Additional
lexical features include BM25 scores computed over control titles and
descriptions after tokenization and stopword removal, Jaccard
coefficient over token sets, and character n-gram overlap measures.
Structural similarity scores derived from shared graph neighborhood
statistics complete this family.

\paragraph{Structural graph features (13 dimensions).}
We construct a heterogeneous control graph with 983 nodes and edges
integrated from the four sources described in
Section~\ref{sec:dataset}. From this graph we extract pairwise
features including shortest-path distance, common-neighbor count,
Jaccard index of neighbor sets, Adamic-Adar index, bridge centrality
scores, degree-product features, and framework-pair indicators. These
features capture topological proximity: controls connected through
multiple paths receive high structural similarity scores regardless of
surface-level text overlap. Figure~\ref{fig:node2vec_umap} visualizes
the graph structure via a UMAP projection of Node2Vec embeddings,
revealing framework-level clustering with cross-framework bridges.

\paragraph{Opus tier scores (2 dimensions).}
A Claude Opus evaluator independently assigns each candidate pair a
tier judgment (0--3) and a confidence score. These two features provide
a calibrated LLM-as-judge signal that complements the embedding-based
similarity features with holistic semantic reasoning.

\subsection{LightGBM Stacker}

The 22-dimensional feature vector is fed to a
LightGBM~\citep{ke2017lightgbm} multiclass classifier that predicts
one of four tiers: \emph{unrelated}~(0), \emph{partial}~(1),
\emph{related}~(2), and \emph{equivalent}~(3). The model uses
\texttt{n\_estimators}${}=500$, learning rate~$0.05$, and
\texttt{min\_samples\_leaf}${}=5$. Inverse-frequency class weights
address the severe class imbalance in the training data.

\subsection{Mondrian Conformal Prediction}

We wrap the LightGBM classifier with a Mondrian conformal predictor
calibrated on a held-out set of $n_{\text{cal}} = 150$ pairs. For each
test instance, the conformal predictor outputs a prediction set
guaranteed to contain the true label with probability
$\geq 1-\alpha$ within each class stratum, providing distribution-free
coverage guarantees~\citep{vovk2005conformal, romano2020malice}.

\subsection{KL-Divergence Router}

A KL-divergence router compares the predicted class distribution against
a uniform reference distribution. Instances with high KL divergence
(confident predictions) are accepted; those with low divergence
(uncertain predictions) are routed to human review. This creates a
natural human-in-the-loop workflow where experts focus on the most
ambiguous cases.
